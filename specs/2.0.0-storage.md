# 2.0.0 — Storage backend

ForgeBox uses a **single PostgreSQL database** for all persistence. There is
no SQLite fallback and no other backend.

## 2.1.0 — Why PostgreSQL only

### 2.1.1 — One store for everything

Tasks, sessions, audit log, users, automations, apps, provider configs, and
the brain knowledge base (pgvector) all live in the same database. One
connection pool, one set of migrations, one backup target.

### 2.1.2 — pgvector is non-optional

The brain feature requires `pgvector` for similarity search over file
embeddings. Since pgvector ships only for PostgreSQL, supporting any other
backend would mean either dropping brain or running a second database.
Neither is acceptable, so PostgreSQL becomes the single source of truth.

### 2.1.3 — Multi-instance gateways

The deployment story (multiple gateway pods behind a load balancer) requires
a network-accessible DB shared across instances. SQLite — the previous
default — could not satisfy this without significant operational pain
(LiteFS, etc.), so it was removed entirely rather than kept as a "single-host
mode."

## 2.2.0 — Configuration

### 2.2.1 — DSN

A single DSN configures the connection. Resolved in order:

1. `FORGEBOX_DATABASE_URL` env var
2. `DATABASE_URL` env var
3. `FORGEBOX_BRAIN_POSTGRES_DSN` env var (legacy, kept for migration grace)
4. `storage.dsn` in the YAML config
5. The default `postgres://forgebox:forgebox@postgres:5432/forgebox?sslmode=disable`

The server refuses to start if the resolved DSN is empty.

### 2.2.2 — Migrations

`postgres.New(dsn)` opens the connection and runs all migrations
idempotently on every boot. Migrations include `CREATE EXTENSION IF NOT
EXISTS vector` and use `CREATE TABLE IF NOT EXISTS` / `ALTER TABLE … ADD
COLUMN IF NOT EXISTS` so re-running is safe. Schema versioning is implicit
through these guards rather than a numbered migration table.

### 2.2.3 — Connection sharing

`internal/storage/postgres.Store` implements both `sdk.StoragePlugin` (core
storage) and `sdk.BrainStore` (brain features). The same `*sql.DB` backs
both. Brain code path receives the same `Store` instance.
