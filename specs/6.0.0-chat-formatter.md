# 6.0.0 Chat Formatter

In-chat rendering of agent natural-language output as basic Markdown.
Hand-rolled parser owned by ForgeBox; no third-party Markdown library.

## 6.1.0 Scope

### 6.1.1 Rendered events

Markdown rendering applies only to `text_delta` events in the task event
stream. `tool_call`, `tool_result`, `tool_pending_approval`,
`tool_approval_resolved`, `error`, `status_update`, `connected`, and
`done` events are not Markdown-rendered.

### 6.1.2 Supported syntax

The renderer supports exactly:

- Inline: `**bold**`, `*italic*`, `` `code` ``, `[text](url)`.
- Block: paragraphs, headings (`#`, `##`, `###`), unordered lists
  (`- `), ordered lists (`1. `), fenced code blocks (```` ``` ````),
  blockquotes (`> `).

Anything outside this set renders as escaped literal text. Raw HTML
never passes through.

## 6.2.0 Safety

### 6.2.1 Output construction

The renderer emits HTML strings only by walking a typed AST. All user
text flows through `escapeHtml` at every leaf before being wrapped in
any tag. The renderer never concatenates raw user text into HTML.

### 6.2.2 URL allowlist

Link `href` values are accepted only when their scheme (after
HTML-entity decoding and whitespace stripping) is `http:`, `https:`, or
`mailto:`, or when the URL is relative or a fragment. Any other scheme
(including `javascript:`, `data:`, `vbscript:`) causes the link to
render as plain escaped text instead of an anchor.

### 6.2.3 Anchor attributes

Generated anchors always carry `rel="noopener noreferrer"` and
`target="_blank"`.

## 6.3.0 Streaming

### 6.3.1 Per-turn buffering

`TaskStream.svelte` folds consecutive `text_delta` events into a single
buffer. A buffer ends — and is rendered — when any non-`text_delta`
event is encountered. On each delta the buffer is re-parsed in full.

### 6.3.2 Partial markers

Inline markers without their closer (e.g. `**bo`) render as literal
text. The same rule applies to `*`, `` ` ``, `[…](…)`, and unclosed
fenced code blocks. Once the closer arrives in a later delta, the
re-parse promotes the literal text to its formatted form.

## 6.4.0 Implementation locations

### 6.4.1 Library

`web/src/lib/utils/markdown/` contains `tokens.ts` (AST types),
`block.ts`, `inline.ts`, `render.ts`, `index.ts` (`renderMarkdown`
entry), and `markdown.test.ts`.

### 6.4.2 Integration point

`web/src/lib/components/TaskStream.svelte` consumes `renderMarkdown`
and emits the result via `{@html}` inside a `.stream__prose` wrapper.

## 6.5.0 Out of scope

Widgets (badges, callouts), images, tables, nested lists, reference
links, autolinks, and HTML passthrough. These can be added in later
chapters if needed.
