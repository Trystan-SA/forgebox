# Anthropic Providers — Design Spec

**Date:** 2026-04-27
**Status:** Approved
**Scope:** Replace the existing `internal/providers/anthropic` package with three
distinct provider classes covering API-key, Claude subscription (OAuth), and local
Claude CLI integrations. Introduce a small shared `pkg/sdk/llmbase` package for
vendor-agnostic primitives (HTTP, SSE, auth strategies) reusable by future providers.

## Overview

ForgeBox needs to talk to Anthropic via three distinct paths, each with different
economics, feature sets, and failure modes:

1. **`anthropic`** — Pay-per-use API key (`sk-ant-api-*`).
2. **`anthropic-subscription`** — Claude Max subscription via OAuth setup token
   (`sk-ant-oat01-*`), generated by `claude setup-token`.
3. **`claude-cli`** — Delegation to a locally installed `claude` binary that owns its
   own credentials and inference loop.

These are not three modes of one provider. They have different billing, different
quotas, different surfaced features, and (for CLI) a fundamentally different
integration shape. Each is a first-class `sdk.ProviderPlugin`. The orchestrator can
route between them by name. They can be configured simultaneously.

The existing `internal/providers/anthropic/anthropic.go` is incomplete (no SSE
parsing, no streaming) and is replaced wholesale.

## Goals

- Three independent `sdk.ProviderPlugin` implementations, registered under separate
  names in `internal/plugins/registry.go`.
- Reuse: `anthropic` and `anthropic-subscription` share an internal Anthropic
  wire-protocol base because they hit the same `/v1/messages` endpoint with the
  same request/response shape.
- A new `pkg/sdk/llmbase` package holding vendor-agnostic primitives (HTTP runner,
  SSE parser, auth strategies, secret-ref resolution) that future provider rewrites
  (OpenAI, Google, Ollama) can adopt.
- Working SSE streaming for the two HTTP-based providers.
- Subscription-mode feature gating that matches Anthropic's actual product limits:
  no 1M context window, no prompt caching, token-only usage (no cost estimation).
- Fail-fast configuration validation: token-prefix mismatches surface at `Init`,
  not at first request.

## Non-Goals (v1)

- Subscription usage tracking (`api/oauth/usage` endpoint, claude.ai web fallback).
  Deferred to a follow-up; the provider returns token counts only.
- OAuth refresh-token flows. The setup-token model used today does not require it.
  An `auth.Refresh()` method is stubbed for future use.
- A unified usage/cost model across providers. Subscription quotas and API-key
  spend are surfaced as-is per provider; reconciliation is a later concern.
- Generalized "vendor CLI delegation" abstraction. Only one CLI provider exists;
  premature abstraction is avoided.

## Architecture

### Layered structure

Three layers. Lower layers are vendor-agnostic; upper layers add vendor specifics.

```
pkg/sdk/llmbase/                      vendor-agnostic primitives
  http.go                             HTTPRunner: POST with retries, status checks
  sse.go                              SSEParser: SSE byte stream -> raw event channel
  streaming.go                        StreamPump: glue HTTPRunner+SSEParser
  auth/
    auth.go                           Auth interface: Apply(*http.Request)
    apikey.go                         APIKeyAuth: parameterized header injection
    oauth.go                          OAuthAuth: Authorization: Bearer + prefix validator
    secretref.go                      env://, file://, exec:// resolution

internal/providers/anthropic/         shared Anthropic wire-protocol code
  base/
    base.go                           AnthropicBase: /v1/messages request/response,
                                      anthropic-version header, SSE event taxonomy,
                                      response parsing, error classification
    betas.go                          APIKeyBetas, OAuthBetas constants, header merge
  models.go                           shared model catalogue

internal/providers/anthropic-api/     pay-per-use
  provider.go                         APIProvider: AnthropicBase + APIKeyAuth
  config.go

internal/providers/anthropic-subscription/   Claude Max
  provider.go                         SubscriptionProvider: AnthropicBase + OAuthAuth
                                      + OAuthBetas + gateRequest() (strips disallowed)
  config.go

internal/providers/claude-cli/        local CLI delegation (no shared base)
  provider.go                         CLIProvider: subprocess driver
  parser.go                           NDJSON envelope -> sdk.StreamEvent
  config.go
  models.go                           claude-cli/* namespace
```

### Composition (Go embedding, not inheritance)

`AnthropicBase` is a struct that holds the HTTP runner, an `auth.Auth` strategy, and
a beta-header set, all supplied at construction. It exposes:

- `buildRequest(*sdk.CompletionRequest) anthropicRequest`
- `buildHeaders() http.Header` (calls `auth.Apply`)
- `parseResponse([]byte) *sdk.CompletionResponse`
- `parseStream(io.Reader) <-chan sdk.StreamEvent` (delegates to `llmbase.StreamPump`)
- `classifyError(*http.Response) error` (maps to `sdk` sentinel errors)
- One overridable hook: `gateRequest(*anthropicRequest)`. Default is no-op;
  the subscription provider overrides to strip disallowed fields.

`APIProvider` and `SubscriptionProvider` each embed `*base.AnthropicBase` and
implement `sdk.ProviderPlugin`. They differ in ~30-50 lines:

```go
// APIProvider construction
b := base.New(base.Options{
    Auth:    auth.NewAPIKey("x-api-key", apiKey),
    Betas:   base.APIKeyBetas,
    BaseURL: cfg.BaseURL,
})
return &APIProvider{AnthropicBase: b}, nil

// SubscriptionProvider construction
b := base.New(base.Options{
    Auth:        auth.NewOAuth(token, "sk-ant-oat"),
    Betas:       base.OAuthBetas,
    BaseURL:     cfg.BaseURL,
    GateRequest: subscriptionGate,   // strips context-1m beta, cache_control fields
})
return &SubscriptionProvider{AnthropicBase: b}, nil
```

`CLIProvider` does not embed `AnthropicBase`. It uses only `llmbase.StreamPump` so
the resulting `sdk.StreamEvent` shape is consistent with the HTTP providers, but
the source is NDJSON over a subprocess pipe, not SSE over HTTP.

### Registry

`internal/plugins/registry.go` `loadBuiltinProvider` adds three entries, replacing
the single existing `"anthropic"` mapping:

```go
case "anthropic":              return anthropic_api.New(), nil
case "anthropic-subscription": return anthropic_subscription.New(), nil
case "claude-cli":             return claudecli.New(), nil
```

## Configuration

Each provider is an independent entry under `providers:`. They can coexist; users
typically configure one or two. All three is supported.

```yaml
providers:
  anthropic:                       # pay-per-use API
    api_key: ${ANTHROPIC_API_KEY}  # literal or secret-ref
    base_url: ""                   # optional override
    timeout_ms: 120000             # optional, default 120000

  anthropic-subscription:          # Claude Max
    token: ${ANTHROPIC_OAUTH_TOKEN}
    base_url: ""
    timeout_ms: 120000

  claude-cli:                      # local CLI delegation
    bin: /usr/local/bin/claude     # optional; defaults to PATH lookup of "claude"
    timeout_ms: 600000             # default 600000 (10 min); CLI runs longer
```

### Secret references

The credential fields (`api_key`, `token`) accept either a literal value or a
secret-reference URI resolved at `Init`:

| Form | Resolution |
|------|------------|
| `sk-ant-api-...` | literal value, used as-is |
| `env://ANTHROPIC_API_KEY` | `os.Getenv("ANTHROPIC_API_KEY")` |
| `file:///etc/secrets/anthropic.token` | `os.ReadFile`, trim whitespace |
| `exec://gcloud auth print-access-token` | `exec.Command`, capture stdout |

Implemented once in `pkg/sdk/llmbase/auth/secretref.go`. Reused by every provider
that takes a credential.

### Validation at `Init()`

All validation runs at startup. No request-time auth surprises.

- **`anthropic`**: resolved `api_key` non-empty, must start with `sk-ant-api-`.
  Reject `sk-ant-oat-*` with: *"this looks like a subscription token; use the
  `anthropic-subscription` provider."*
- **`anthropic-subscription`**: resolved `token` non-empty, must start with
  `sk-ant-oat`. Reject `sk-ant-api-*` with the symmetric message.
- **`claude-cli`**: resolve `bin` (or `exec.LookPath("claude")`), `os.Stat` it,
  ensure executable. Run `claude --version` once; non-zero exit fails `Init`.
  Do **not** validate credentials — `claude` owns that lifecycle.

## Request flow

### `anthropic` and `anthropic-subscription` (shared path)

```
Provider.Complete(ctx, req)
  -> base.buildRequest(req)              convert sdk.CompletionRequest -> /v1/messages
  -> base.gateRequest(payload)           subclass hook (subscription strips fields)
  -> base.buildHeaders()
       anthropic-version: 2023-06-01
       Content-Type: application/json
       anthropic-beta: <subclass beta set>
       <auth.Apply -- x-api-key OR Authorization: Bearer>
  -> llmbase.HTTPRunner.Do
  -> 200: base.parseResponse
  -> non-2xx: base.classifyError
```

`Stream` follows the same path; the response body is fed to `llmbase.SSEParser` and
`StreamPump`, emitting `sdk.StreamEvent` values for the Anthropic event taxonomy:
`message_start`, `content_block_start`, `content_block_delta` (text or tool_use
input JSON deltas), `content_block_stop`, `message_delta`, `message_stop`.

### `claude-cli`

```
CLIProvider.Complete(ctx, req)
  -> mapModelID(req.Model)               "claude-sonnet-4-6" -> "claude-cli/claude-sonnet-4-6"
  -> exec.CommandContext(ctx, p.bin,
        "-p", "--output-format", "stream-json", "--verbose",
        "--permission-mode", "bypassPermissions")
  -> write prompt to stdin, close stdin
  -> scan stdout NDJSON line-by-line
  -> map each envelope to sdk.StreamEvent
  -> non-zero exit: sdk.ErrCLIBackend wrapping stderr tail
```

The CLI provider does not authenticate on our side. The `claude` binary reads its
own credentials from `~/.claude` of the gateway process user. Operators are
responsible for ensuring those credentials exist; the spec documents this clearly.

## OAuth gating rules (subscription only)

Subscription auth has product-level restrictions enforced by Anthropic's API.
We mirror those restrictions on the client side so misconfigured callers get a
working response (with a logged warning) rather than a hard 4xx.

Centralized in `subscriptionGate(*anthropicRequest)` on the subscription provider.

| Rule | Behavior |
|------|----------|
| 1M context window | If the caller adds the `context-1m-2025-08-07` beta or selects a 1M model variant, strip the beta. Log once per request at WARN: *"subscription auth does not support 1M context; falling back to 200K"*. |
| Prompt caching | If any message contains `cache_control` blocks, strip them in place. Log once per request at WARN: *"prompt caching not supported on subscription auth; ignoring cache_control"*. |
| Cost estimation | `Usage` carries `InputTokens` and `OutputTokens` only. `EstimatedCostUSD` is left zero; downstream cost-tracking treats subscription tokens as free at request time. |

Logs use `slog` per `CLAUDE.md`.

### Beta header sets

```go
// internal/providers/anthropic/base/betas.go
var APIKeyBetas = []string{
    "fine-grained-tool-streaming-2025-05-14",
    "interleaved-thinking-2025-05-14",
}

var OAuthBetas = []string{
    "claude-code-20250219",
    "oauth-2025-04-20",
    "fine-grained-tool-streaming-2025-05-14",
    "interleaved-thinking-2025-05-14",
}
```

`mergeAnthropicBetaHeader` deduplicates against any caller-supplied
`anthropic-beta` value before sending.

## Error classification

Shared in `anthropic/base/base.go`. Maps HTTP status and signal conditions to
`sdk` sentinel errors so the orchestrator can branch without parsing strings.

| HTTP / signal | sdk error |
|---|---|
| 401, 403 | `sdk.ErrAuth` — invalid token, missing scope |
| 429 | `sdk.ErrRateLimit` — also returned for subscription quota exhaustion; response body preserved on the wrapped error for surfacing in UI |
| 400 with `context_length_exceeded` | `sdk.ErrInputTooLarge` |
| 5xx | `sdk.ErrTransient` — caller may retry |
| Network error, context deadline | `sdk.ErrTransient` |
| CLI non-zero exit | `sdk.ErrCLIBackend` (new) — wraps stderr tail |

`pkg/sdk/errors.go` gains these sentinel values. No interface changes to
`sdk.ProviderPlugin`.

## Models

### `anthropic` and `anthropic-subscription`

Share the same catalogue (defined once in `internal/providers/anthropic/models.go`):

- `claude-sonnet-4-6` (200K in / 16K out, tools, vision)
- `claude-haiku-4-5-20251001` (200K in / 8K out, tools, vision)
- `claude-opus-4-7` (200K in / 16K out, tools, vision)

The 1M-context variant is not advertised on subscription. If/when the catalogue
gains a 1M variant, `anthropic-subscription` filters it from `Models()`.

### `claude-cli`

Mirrors the same models but in the `claude-cli/` namespace:

- `claude-cli/claude-sonnet-4-6`
- `claude-cli/claude-haiku-4-5-20251001`
- `claude-cli/claude-opus-4-7`

`mapModelID` strips the `claude-cli/` prefix when invoking the binary's `--model`
flag, since `claude` itself uses bare model IDs.

Per `CLAUDE.md`: table-driven, `testify/require`, `t.Helper()` in helpers.
Mocks via interfaces. White-box (same package).

```

  sse_test.go               fixture SSE streams: text deltas, tool_use, partial chunks
  auth/apikey_test.go       header injection, parameterized header name
  auth/oauth_test.go        bearer header, prefix validator
  auth/secretref_test.go    env/file/exec resolution, malformed refs

internal/providers/anthropic/base/
  base_test.go              request build, response parse, error classification table
  betas_test.go             merge dedup behavior

internal/providers/anthropic-api/
  provider_test.go          init validation (rejects oat tokens), header set,
                            integration against httptest.Server

internal/providers/anthropic-subscription/
  provider_test.go          init validation (rejects api tokens), oauth headers,
                            gating: context-1m beta stripped, cache_control stripped,
                            warning logged, request still succeeds

internal/providers/claude-cli/
  provider_test.go          fake binary at testdata/fake-claude.sh emitting canned
                            NDJSON; verifies parsing, stderr surfacing, ctx timeout
```

Integration tests use `//go:build integration` and an `httptest.Server` replaying
fixture responses — no live Anthropic calls. CLI tests ship a shell-script fake
so the real `claude` binary is not required in CI.

## Migration

The current single `anthropic` provider is replaced. Existing configs of the form

```yaml
providers:
  anthropic:
    api_key: sk-ant-api-...
```

continue to work unchanged — the new `anthropic` provider accepts the same key.
Configs using `sk-ant-oat-*` against the old provider would never have worked
(it sent `x-api-key`); they now must move to `anthropic-subscription`. The
`Init` rejection message tells operators exactly what to change.

No database migrations. No API changes visible to the orchestrator beyond the
new sentinel errors and two new provider names appearing in `ListProviders()`.
